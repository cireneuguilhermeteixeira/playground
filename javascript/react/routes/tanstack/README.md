# TanStack Router File-Based Routing Playground

This project is a practical playground for **TanStack Router** with **file-based routing** using **React + Vite + TypeScript**.

It demonstrates how route files map to URLs, including:

- static routes
- nested layouts
- index routes
- dynamic params
- pathless layouts
- escaping route tokens in filenames
- ignoring files/folders from route generation

## Tech Stack

- React 18
- TypeScript
- Vite 5
- `@tanstack/react-router`
- `@tanstack/router-plugin` (Vite plugin that generates the route tree)
- `@tanstack/react-query` (used for client-side async data fetching and caching)

## How Routing Is Wired

### 1. Router plugin in Vite

`vite.config.ts` enables the TanStack router plugin:

- `routesDirectory: './src/routes'`
- `generatedRouteTree: './src/routeTree.gen.ts'`
- `routeFileIgnorePrefix: '-'`

This means route files are read from `src/routes`, and the generated tree is written to `src/routeTree.gen.ts`.

### 2. Router creation

`src/router.tsx` creates the router from `routeTree.gen.ts` and registers types with module augmentation.

### 3. App bootstrap

`src/main.tsx` renders:

- `<QueryClientProvider client={queryClient}>`
- `<RouterProvider router={router} />`

`src/queryClient.ts` centralizes QueryClient configuration (`staleTime`, focus refetch behavior, etc.).

## Layout and Styling

- `src/routes/__root.tsx` defines the root layout with:
  - app shell (`.app` + `.card`)
  - top navigation links
  - `<Outlet />` for child routes
- `src/App.css` styles the shell, nav, and active link states.
- `src/index.css` defines base typography and global resets.

## File-Based Routing Conventions Demonstrated

### Static route

- `src/routes/about.tsx` -> `/about`

### Nested route tree with layout

- `src/routes/posts.tsx` -> `/posts` layout (renders `<Outlet />`)
- `src/routes/posts.index.tsx` -> index page for `/posts`

### Dynamic params

- `src/routes/posts.$postId.tsx` -> `/posts/$postId`
- `src/routes/posts.$postId.edit.tsx` -> `/posts/$postId/edit`
- `src/routes/orgs.$orgId.users.$userId.tsx` -> `/orgs/$orgId/users/$userId`

These pages use `Route.useParams()` to read URL params.

### React Query route example

- `src/routes/posts.query.tsx` -> `/posts/query`

This page demonstrates:

- data loading with `useQuery`
- loading, error, and success UI states
- caching by `queryKey`
- manual refetch with a button

The query function is in:

- `src/routes/posts/-helpers/fetchPostsQueryExample.ts`

### Pathless layout (underscore prefix)

- `src/routes/_admin.tsx` defines a layout that **does not add** `_admin` to the URL.
- `src/routes/_admin.users.tsx` maps to `/users` but renders inside `_admin` layout.
- `src/routes/_admin.settings.profile.tsx` maps to `/settings/profile` under the same pathless layout.

### Escaped filename token

- `src/routes/[index].tsx` maps to `/index`.

Square brackets let the filename represent a literal segment safely.

### Ignored route files (hyphen prefix)

Because `routeFileIgnorePrefix` is `-`:

- `src/routes/-internal/not-a-route.tsx` is **not** treated as a route.
- Helper/component files under `src/routes/posts/-helpers` and `src/routes/posts/-components` are also ignored by route generation.

## Shared Route Utilities

- `src/routes/posts/-helpers/formatTag.ts`
  - simple helper that formats a tag to uppercase with a `#` prefix.
- `src/routes/posts/-components/PostSummary.tsx`
  - reusable UI component for post summary rendering.

`posts.$postId.tsx` demonstrates importing and using both shared files from ignored folders.

## Generated Files

- `src/routeTree.gen.ts` is auto-generated by TanStack router plugin.
- You should not manually maintain generated route definitions.

## Current Goal of This Repository

This repository is focused on learning and validating **TanStack Router file-based routing patterns** with clear, isolated examples for each naming convention.
