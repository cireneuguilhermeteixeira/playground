# React Window Pagination Demo

This project shows a simple way to use `react-window` in a React app.

The goal is to render a large list without putting every item into the DOM at the same time. Instead of rendering thousands of rows, `react-window` only renders the rows that are visible inside the scroll area, plus a small extra buffer.

## Why `react-window` exists

Rendering a very large list with normal `.map()` can become expensive because the browser has to manage too many DOM nodes.

`react-window` solves this by using **windowing** or **virtualization**:

- The full dataset still exists in memory.
- The UI only renders the rows currently visible on screen.
- When the user scrolls, old rows are removed and new rows are mounted.

This keeps scrolling smoother and reduces DOM work.

## How it works in this project

The app creates a dataset with `10,000` items and passes the current page slice to `react-window`.

The main piece is the `List` component:

```tsx
<List
  rowComponent={Row}
  rowCount={currentItems.length}
  rowHeight={ROW_HEIGHT}
  rowProps={{ items: currentItems }}
  style={{ height: LIST_HEIGHT }}
/>
```

Here is what each prop does:

- `rowComponent`: the React component responsible for rendering one row.
- `rowCount`: how many rows exist in the current list.
- `rowHeight`: the fixed height of each row.
- `rowProps`: extra data passed to each row.
- `style.height`: the visible area height. This defines how many rows can appear at once.

## The row component

`react-window` calls the row component for the visible indexes only.

In this project, the row receives:

- `index`: the row position inside the current page.
- `style`: inline positioning styles required by `react-window`.
- `items`: the current page data.
- `ariaAttributes`: accessibility attributes generated by the library.

Simplified version:

```tsx
function Row({ index, style, items, ariaAttributes }: RowComponentProps<RowProps>) {
  const item = items[index]

  return (
    <article style={style} {...ariaAttributes}>
      <h3>{item.title}</h3>
    </article>
  )
}
```

The `style` prop is mandatory. Without it, rows will not be positioned correctly.

## Pagination + virtualization

This example uses two layers:

1. Pagination chooses which slice of the dataset is active.
2. `react-window` virtualizes only the rows inside that current slice.

That means:

- The app does not display all `10,000` items at once.
- It displays one page at a time.
- Even inside that page, it still does not render every row in the DOM simultaneously.

So pagination controls the data range, and `react-window` controls DOM performance.

## Mental model

You can think about it like this:

- `dataset`: the complete source data.
- `currentItems`: the current page selected by the pagination controls.
- `List`: a viewport over `currentItems`.
- `Row`: the template used to paint each visible item.

## When to use `react-window`

It is a good fit when:

- you have many rows
- each row has a predictable height
- rendering everything at once is too expensive

It is usually unnecessary for very small lists.

```bash
npm install

```

## Summary

`react-window` does not reduce the size of your data. It reduces how much of that data becomes DOM at the same time.

That is the main idea behind this demo.
